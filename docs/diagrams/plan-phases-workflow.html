job-market-agent · PLAN.md Phased Workflow

7 phases · Phase 1 = current vertical slice · each phase adds one capability without re-architecting

What ships at each phase

  • • P0 · Job schema with all v1 fields
  • • P1 ◆ current · jma crawl · TesterHome → SQLite
  • • P2 · Randstad + Bing + Playwright + sources status
  • • P3 · DeepSeek extraction + canonical skills
  • • P4 · market.md (stats + one narrate call)
  • • P5 · fit.md (persona + score + top-30 refine)
  • • P6 · jma run + cross-run delta + README

Why this phasing

  • • User mandate: job-search works end-to-end before anything else
  • • Each phase exits on a demonstrable real-case (PLAN section 4)
  • • runs table lands at P1 so P6 trend = SQL query, not migration
  • • LLM enters only at P3 — pure pipeline before that
  • • Per-prompt sha1 cache from P3 keeps re-runs free across P4 / P5
  • • TDD red-green-refactor inside every phase

LLM cost ceilings (PLAN cases)

  • • Extract: ≈ $0.0005 / job · 300 jobs ≈ $0.15 (one-time, cached)
  • • Narrate market: ≈ $0.002 / report · cached by stats digest
  • • Persona: 1 call / profile · cached by profile hash
  • • Fit refine: 1 call / report over top-30 only
  • • Architectural guarantee: no LLM does arithmetic — pandas does
  • • Cache hit → re-runs are $0 across the board
