jma — Database Schema

SQLite (WAL) at data/jobs.db + gzipped raw HTML blobs on disk — built by src/jma/storage/db.py

Dedup model (ADR-0001)

  • • id = sha1("source:internal_id") — per-source JobObservation
  • • canonical_id = sha1(normalize(title|company|city)) — cross-source Job
  • • Same Job seen via two sources → two rows, shared canonical_id
  • • Aggregations over "Jobs" MUST `GROUP BY canonical_id`

Run lifecycle (ADR-0002)

  • • Every `jma crawl` = one new runs row (uuid hex)
  • • Same (region, keywords) twice = two Runs
  • • pipeline.crawl.run always calls finish_run (even on exception)
  • • No row left with finished_at IS NULL

URL freshness (ADR-0003)

  • • jobs.url_status: live / gone / unknown
  • • 200 → LIVE · 404/410 → GONE · others → preserve
  • • UPSERT keeps earned signal: 'unknown' never clobbers 'live'/'gone'
  • • url_cache.blob_ref + jobs.raw_payload_ref both point at the same .html.gz files
