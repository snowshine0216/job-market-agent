job-market-agent · Current Implementation Workflow

Phase 0 + Phase 1 vertical slice — TesterHome crawl → SQLite. Async httpx, pure domain, idempotent storage.

End-to-end flow (numbered above)

  • • 1. CLI parses args → pipeline loads testerhome.yaml
  • • 2. URL cache lookup (24h SQLite-backed)
  • • 3. On miss: httpx fetch with retry / backoff
  • • 4. blockage.classify → ok / blocked / empty / rate-limited
  • • 5. parser yields list items + gzip raw to blobs/
  • • 6. normalize parses salary / exp / location
  • • 7. dedup.job_id → sha1 source-scoped id
  • • 8. insert_jobs upserts; finish_run closes the run row

Pure / I/O boundary discipline

  • • domain/* is pure: deterministic, no network, no mutation
  • • sources/*, storage/*, pipeline/* own all I/O
  • • Pydantic v2 models are frozen — no in-place mutation
  • • All v1 Job fields exist up-front with safe defaults
  • • Phase 3+ fills empty fields; schema never alters

What's NOT in this slice yet

  • • Randstad, Bing, Playwright sources (Phase 2)
  • • LLM client, extraction, narration (Phase 3+)
  • • Skills canonicalisation / skills.yaml (Phase 3)
  • • Market & fit reports, persona, scoring (Phase 4–5)
  • • jma run, sources status, cross-run delta (Phase 2/6)
