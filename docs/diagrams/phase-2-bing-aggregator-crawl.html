job-market-agent · Phase 2 — Bing Aggregator Crawl

SerpAPI Bing engine · 5 boards via site: queries · snippet-only Jobs · per-Run .json.gz blobs · source=bing:<host> (ADR-0005)

End-to-end flow (numbered above)

  • • 1. cli.py loads config/sources/bing.yaml, checks SERPAPI_KEY env, builds source factory
  • • 2. pipeline.crawl.run opens DB, calls start_run, invokes source.crawl()
  • • 3. Per page: cache_get → on miss http.get(serpapi_url) → on_fetch writes url_cache + .json.gz blob
  • • 4. Parse organic_results: drop off-target hosts · map snippet → parse_salary/experience
  • • 5. dedup.job_id (source-scoped) + dedup.canonical_id (cross-source merge key, ADR-0001)
  • • 6. insert_jobs writes jobs row + run_jobs.raw_payload_ref (per-Run blob snapshot)
  • • 7. finish_run always (ADR-0002) — even on unhandled exception

Phase 2 invariants (ADR-0005)

  • • source = "bing:<host>" where host = matched target_sites entry (subdomain-collapsed)
  • • data_quality = 0.4 for all Phase 2 Bing rows (snippet-only baseline)
  • • raw_payload_ref stored twice: jobs (latest-seen) + run_jobs (per-Run snapshot)
  • • Location.city = None in Phase 2 — no best-guess from SERP snippet
  • • Per-host URL regex (id_patterns) allowed; per-host snippet regex forbidden
  • • Company extraction is heuristic-only: generic delim-split + site_names anchor
  • • No detail-fetch in Phase 2 — deferred to Phase 2.1 (see ADR-0005 trigger)

Blob layout + two-level raw_payload_ref

  • • One .json.gz blob per SerpAPI page — N jobs share it
  • • Path: data/raw/bing/{yyyymmdd}/{sha1(url)[:16]}.json.gz
  • • run_jobs.raw_payload_ref — blob captured during that Run (used by jma view --run <old>)
  • • jobs.raw_payload_ref — latest-seen blob (used by aggregation-level reads)
  • • jma view renders file://{data_root}/{raw_payload_ref} as a clickable link

What's NOT in Phase 2

  • • Detail-fetch / direct JD page crawl (deferred — ADR-0005 Phase 2.1 trigger)
  • • LLM extraction (Phase 3 — description_text stores raw snippet as input)
  • • Skills canonicalisation / skills.yaml (Phase 3)
  • • Market & fit reports, persona, scoring (Phase 4–5)
  • • Randstad / Playwright / direct BOSS Zhipin source (deferred)
